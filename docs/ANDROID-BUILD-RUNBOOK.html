1 Install Dependencies

Open a terminal in the MNC/ folder and run:

npm install

This pulls in @capacitor/core, @capacitor/cli, @capacitor/android, and the splash-screen, status-bar, and push-notifications plugins.

2 Add the Android Platform

npx cap add android

This creates the android/ folder with a full Gradle project. You only run this once.

Heads-up: If you ever re-run this and it says "android already exists," skip to Step 3.

3 Sync Web Files Into the Native Project

npx cap sync android

This copies your web files (index.html, sw.js, images/, etc.) into android/app/src/main/assets/public/ and installs native plugin code. Run this every time you change web code.

4 Add Splash Screen Assets

Capacitor’s splash screen needs a PNG or 9-patch in the Android drawable folders. The simplest approach:

  1. Take your images/mncLogo-1024.webp and convert it to PNG (any image tool or online converter).
  2. Name it splash.png.
  3. Place it in android/app/src/main/res/drawable/splash.png.
  4. Optionally add sized versions in drawable-mdpi, drawable-hdpi, drawable-xhdpi, drawable-xxhdpi, and drawable-xxxhdpi for best results.
Background color: The splash background is already set to #FAF5F7 in capacitor.config.json. The logo will be centered over it.

5 Set the App Icon

Replace the default Capacitor launcher icon with the MNC logo:

  1. In Android Studio, right-click android/app/src/main/res → New → Image Asset.
  2. Set Icon Type to Launcher Icons (Adaptive & Legacy).
  3. For Foreground Layer, pick your mncLogo-1024.webp (convert to PNG first if needed).
  4. Set Background Layer color to #FAF5F7.
  5. Click Next → Finish. Android Studio generates all density versions automatically.

6 Open in Android Studio

npx cap open android

This opens the android/ folder in Android Studio. Let Gradle sync finish (progress bar at the bottom). This can take a minute or two on the first run.

7 Build a Debug APK (Test on Your Phone)

In Android Studio:

  1. Wait for Gradle sync to complete (bottom status bar).
  2. Connect your Android phone via USB with USB Debugging enabled (Settings → Developer Options → USB Debugging).
  3. Select your device in the toolbar dropdown next to the green play button.
  4. Click the green play button (▶) or press Shift+F10.
  5. The app will build, install, and launch on your phone.
Alternatively, generate an APK file you can share:
Build → Build Bundle(s) / APK(s) → Build APK(s)
The APK will be at android/app/build/outputs/apk/debug/app-debug.apk.

8 Build a Release AAB (for Google Play)

Google Play requires an Android App Bundle (.aab), not a plain APK.

8a — Create a Signing Key (one time)

# Run in terminal — remember this password!
keytool -genkey -v \
  -keystore mnc-release-key.jks \
  -keyalg RSA -keysize 2048 \
  -validity 10000 \
  -alias mnc

Store mnc-release-key.jks somewhere safe (not in the repo). You’ll need it for every future release.

8b — Configure Signing in Gradle

Open android/app/build.gradle and add inside android { }:

signingConfigs {
    release {
        storeFile file('../../mnc-release-key.jks')
        storePassword 'YOUR_STORE_PASSWORD'
        keyAlias 'mnc'
        keyPassword 'YOUR_KEY_PASSWORD'
    }
}
buildTypes {
    release {
        signingConfig signingConfigs.release
        minifyEnabled false
        proguardFiles getDefaultProguardFile(
            'proguard-android-optimize.txt'),
            'proguard-rules.pro'
    }
}
Security tip: For production, move passwords to local.properties or environment variables instead of hard-coding them in build.gradle.

8c — Build the AAB

In Android Studio: Build → Generate Signed Bundle / APK → Android App Bundle → Next. Select your keystore, enter passwords, pick release, and finish.

Or from the terminal:

cd android
./gradlew bundleRelease

Output: android/app/build/outputs/bundle/release/app-release.aab

9 Upload to Google Play Console

  1. Go to play.google.com/console and create a new app.
  2. Fill in the store listing: app name (My Nail Connection), short description, full description, screenshots, feature graphic.
  3. Under Release → Production (or start with Internal Testing for a quick test), upload your .aab file.
  4. Complete the content rating questionnaire, set pricing to Free, and select target countries.
  5. Submit for review.
Pro tip: Start with Internal Testing track first. You can add up to 100 testers by email — no review wait, instant installs via a link.

Everyday Workflow (After Initial Setup)

When you update the web code (index.html, etc.), the update cycle is just two commands:

# 1. Copy latest web files into the Android project
npx cap sync android

# 2. Open Android Studio and hit play (or build APK/AAB)
npx cap open android

Troubleshooting

White screen on launch

Usually means web files didn’t sync. Run npx cap sync android again and check that android/app/src/main/assets/public/index.html exists.

Gradle sync fails

Make sure Android Studio has SDK 34 installed. File → Settings → Languages & Frameworks → Android SDK → SDK Platforms — check Android 14 (API 34).

“App not installed” on device

Usually a signing conflict. Uninstall the old version from the device first, then install fresh.

Service Worker / offline issues

Capacitor serves files from https://localhost by default (set via androidScheme: "https" in config). Your existing sw.js should work. If caching causes stale content, bump your SW version or clear app data.

Status bar color not applying

The StatusBar plugin is configured in capacitor.config.json. If it’s still not working after sync, you can also set it programmatically in index.html:

if (window.Capacitor && window.Capacitor.Plugins.StatusBar) {
  window.Capacitor.Plugins.StatusBar.setBackgroundColor({ color: '#C4786A' });
  window.Capacitor.Plugins.StatusBar.setStyle({ style: 'LIGHT' });
}
